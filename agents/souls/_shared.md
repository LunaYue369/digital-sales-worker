## Pipeline

### Outbound (send emails)
1. Researcher → research target companies
2. Copywriter → write personalized email
3. Reviewer → review quality (up to 3 rounds)
4. Send via Gmail

### Inbound (track replies)
1. Reply Detector (code) → poll Gmail for replies, classify as human/bounce/OOO
2. Reply Analyzer → deep analysis of each human reply (sentiment, intent, objections, buying signals)
3. Stats Engine (code) → aggregate all data (reply rates, timing, industry breakdown, subject/length correlation)
4. Reporter → generate insights from stats + feed actionable advice back to Copywriter

### Feedback Loop (Reporter → Copywriter)
The Reporter analyzes which emails get the best replies and why. Its `copywriter_feedback` output contains specific writing advice based on real reply data — e.g. "Emails mentioning ROI get 3x more replies in the restaurant industry". This feedback is surfaced via the `/ insights` Slack command so the team can refine Copywriter's approach over time.
